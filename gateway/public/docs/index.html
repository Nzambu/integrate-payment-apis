    Introduction

    Sample microservices architecture integrating external application programming interfaces

    This documentation aims to provide all the information you need to work with our API.

    As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

    Base URL

    http://localhost/microservice/

    Authenticating requests

    This API is not authenticated.

    Authentication

    Only authenticated users can make the get, post, put, patch and delete requests using the endpoints provided in this section. Integration into different languages is shown using relevant example requests. Follow the examples as a guide showing how to provide the parameters in the right place.

    Authentication - Login

    Users sign up or log in by making a post request with user email and password. The endpoint is open to the public to aid users gain access to their accounts. The login credentials are explained in the body section below. Integration in sample languages is shown with example requests. The desired response is as shown in example response 200.

    Example request:
    curl --request POST \
    "http://localhost/microservice/api/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"et\",
    \"password\": \"quia\"
}"

    const url = new URL(
    "http://localhost/microservice/api/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "et",
    "password": "quia"
}

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

    Request      

    POST api/register

    Body Parameters

    first_name  string  

    The first name of the user. Must not be greater than 45 characters.

    middle_name  string optional  

    The middle name of the user. Must not be greater than 45 characters.

    last_name  string  

    The last name of the user. Must not be greater than 45 characters.

    email  string  

    The primary email for the user's account. Must be a valid email address.

    password  string  

    The secret word for user's account. Must be at least 8 characters.

    password_confirmation  string  

    A confirmation of the secret word for the user's account. Must be at least 8 characters.
